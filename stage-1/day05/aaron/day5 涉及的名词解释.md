# 比特币脚本（Bitcoin Script）

它是一个基于堆栈的编程，对比特币交易进行一种锁定，解锁需要特定的解锁脚本。它可以用于描述比特币在什么样特定的条件下可以进行交易。
原理：由数据（公钥&签名）和操作码组成，栈顶元素为 1 的脚本才算有效，每个输出有一个锁定脚本，必须提供解锁脚本才能花费该输出。


# taproot升级
Schnorr 签名(BIP 340)、Taproot (BIP 341)和TapScript (BIP 342)
-是三个BIPS的汇编，类似聚合签名，只破解一个签名没有意义，需要3个一起破解

#  Merkel 抽象语法树(MAST)

把脚本不同的分支做成一棵 Merkle 树，链上只承诺这棵树的根；真正花钱时，只公开你用到的那一条分支和它的证明。
好处是无论交易背后是多么复杂的签名，链上的记录和普通交易没有区别，实现了隐私最大化

# 铭文
-将图片、文本、代码等元数据，写进BTC的最小单位“聪”里，形成数字资产

### 特点
随聪转移：铭文和那枚聪绑定，转账即转“作品”。
费用与拥堵：内容越大，手续费越高，会占用区块空间。


# 符文
是在比特币上发行/转账“可替代代币”的一种新协议（由 Ordinals 作者 Casey 提出，2024 年上线）。不依赖铭文文本，走更“原生 UTXO”的记账方式。

### 流程
Etch（刻写）：创建新代币，定义名称、精度、小数位、供应/铸造规则等。
Mint（铸造）：按规则增发。
Transfer（交易）：把某 UTXO 上的代币余额转到新 UTXO.

# PSBT
Partially Signed Bitcoin Transactions (PSBT)
包含了所有的输入、输出信息，但缺少必要的签名。它可以在各方之间传递，每个拥有私钥的参与者添加自己的部分签名，最后，当收集到足够的签名后，就可以组合成一个完全有效的、可以广播到网络的比特币交易。

### 流程
1. 创建PSBT.包含UTXO，但未包含签名
2. 添加签名
3. 合并签名
4. 最终化交易，被广播到比特币网络里

# RGB协议
RGB客户端验证 ，加强隐私。在比特币网络上只能看到是一笔从1到2的交易，但是产生这笔交易的细节是不会在公开记录里披露的，在RGB协议里能做出更多细节的约定，这些约定只有验证与验证的对象才能看到，既保护了用户隐私，也不需要比特币网络处理这类复杂的协议，从而增加了网络自身的扩展性
