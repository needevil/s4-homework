# DAY02 作业

## 名词解释

### web3：

web3是区块链技术的上层应用，基于区块链等去中心化技术，重构互联网的所有权、数据控制权和价值分配方式。

### 区块链：

分布式账本技术，核心是通过去中心化、加密验证实现数据不可篡改与可追溯。

### 区块头：

区块头包含了本区块的hash值，父节点的hash值，本区块的Nonce，本区块的merkle root信息，时间戳信息

### 区块体：

区块体包含了区块的交易，通过merkle tree运算得到区块头里面的merkle root hash

### 哈希算法

是一种将任意长度的输入数据，通过特定计算规则转换为固定长度输出的函数，输出结果称为 “哈希值”Hash Value，它是不可逆的

### Nonce值

Number used once，意为 “仅使用一次的数字”，是一个临时生成的随机数或计数器，核心作用是确保数据的唯一性、安全性或满足特定计算规则，且通常只被使用一次

### Merkle tree

二叉树结构。验证根节点的hash值就可以验证大量数据是否被篡改，验证叶子节点的hash以及哈希路径就可以看是否属于本merkle tree

### 区块链的特性

#### 去中心化：

区块链本质是一份公开的流水账，记载着每个节点过往的交易历史，公开程度堪比居委会的“公开栏”。它存在每个节点里。如果某个用户想进行一笔交易，他必须把交易的细节公布出来，这个公布的过程叫广播。收到广播的每个节点都要对交易申请进行审核投票，然后根据投票结果修改公共账本。

#### 不可篡改性

如果想篡改一个区块，它的哈希值也会变化，这就意味着它的下一个区块也需要修改，否则就对不上号。以此类推，后面所有新生成的区块都要修改。工作量非常大，基本上不太可能。

#### 可追溯性

在分布式账本中，每一笔交易都会被记录下来，所以具有绝对的可追溯性。
